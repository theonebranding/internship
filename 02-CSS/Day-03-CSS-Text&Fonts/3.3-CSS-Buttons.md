## ðŸŽ¨ Adding Hover, Active, and Focus States

These are **pseudo-classes** â€” special CSS selectors that apply styles based on user interaction.

| Pseudo-Class | Description                                  |
| ------------ | -------------------------------------------- |
| `:hover`     | When mouse pointer is over the element       |
| `:active`    | When button is being clicked                 |
| `:focus`     | When button is focused (e.g., using Tab key) |

### Example

```css

  background-color: #0056b3;
}

button:active {
  background-color: #003f7f;
}

button:focus {
  outline: 2px solid #ffae00;
  outline-offset: 2px;
}
```

Now your button responds when hovered, clicked, or focused â€” making it feel alive.

## ðŸ§© The Anatomy of a Link

A link is created using the `<a>` tag (short for *anchor*).

```html
<a href="https://www.google.com">Visit Google</a>
```

* `href` â†’ defines the linkâ€™s destination (URL).
* The text between the tags is the clickable part.

By default:

* Unvisited links are **blue**.
* Visited links are **purple**.
* Active (being clicked) links are **red**.
* Hovered links are **underlined**.

We can control all that with CSS pseudo-classes.

---

## ðŸŽ¨ CSS Link States

| Pseudo-Class | Description                       |
| ------------ | --------------------------------- |
| `a:link`     | Unvisited link                    |
| `a:visited`  | Link that has been clicked before |
| `a:hover`    | When the mouse hovers over it     |
| `a:active`   | When itâ€™s being clicked           |

**Order matters!**
Always define them in this order to avoid conflicts:

> `a:link â†’ a:visited â†’ a:hover â†’ a:active`
