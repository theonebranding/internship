# ğŸ¯ **4.3.1 â€” What Are Events? (Simple Explanation)**

An **event** is an **action** happening in the browser.

Examples:

* A user **clicks** a button
* A user **types** in input
* A user **hovers** over image
* A form gets **submitted**
* The page has **loaded**
* A user scrolls

JS can **detect these events** and run functions in response.

# ğŸŸ¦ **4.3.3 â€” 3 Ways to Attach Events**

---

# **Way 1: Inline Event (NOT recommended)**

```html
<button onclick="hello()">Click</button>

<script>
function hello() {
  alert("Hello");
}
</script>
```

Works, but bad practice.

---

# **Way 2: Event Property (Simple)**

```html
<button id="btn">Click</button>

<script>
let b = document.getElementById("btn");
b.onclick = function() {
  console.log("Button clicked");
}
</script>
```

Easy but **overwrites** previous events.

---

# **Way 3: addEventListener() (BEST â€” MOST USED)**

```js
b.addEventListener("click", function() {
  console.log("Clicked using addEventListener");
});
```

Benefits:

* Can attach multiple listeners
* More control
* Can remove listeners
* Perfect for modern JS
